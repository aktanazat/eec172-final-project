Project Description

AEGIS-172 is an interactive tabletop defense arena built around the CC3200 LaunchPad and an Arduino UNO coprocessor. A stepper-driven ultrasonic radar sweeps the environment, identifying threat sectors through fused signals from ultrasonic distance, ambient light change, and tilt events. The defender controls a servo shield and tactical actions through a joystick and IR remote, while a second player influences round pressure through attack modes triggered by remote buttons.

Each match is cloud-adaptive. The CC3200 requests a mission profile from AWS IoT Core, Lambda generates wave parameters, and S3 stores mission files and replay artifacts. At round completion, Lambda computes a weighted score with latency penalties and writes results to a leaderboard, allowing reproducible verification of round behavior.

The system boots standalone from flash with no host terminal connection. All interaction is through the joystick, IR remote, and on-device displays (128x128 OLED radar HUD and LCD1602 score readout). The CC3200 handles the state machine, cloud communication, OLED rendering, and IR decoding. The UNO handles stepper timing, ultrasonic sampling, servo actuation, RGB/buzzer feedback, and joystick analog reads, communicating over UART at 9600 baud using a structured frame protocol.

Hardware Protocols

  • SPI -- CC3200 to OLED (SSD1351)
  • I2C -- CC3200 to BMA222 accelerometer
  • UART -- CC3200 to Arduino UNO (9600 baud)

Web Service

  • AWS IoT Core -- device shadow, HTTPS REST
  • AWS Lambda -- mission generation, round scoring
  • AWS S3 -- mission files, replays, leaderboard

Sensing Devices

  • HC-SR04 -- ultrasonic distance (threat detection)
  • IR receiver -- remote control input
  • Photoresistor -- ambient light anomaly
  • BMA222 -- tilt/vibration
  • Joystick -- analog shield positioning

Design

State Machine

The match engine runs an eight-stage state machine. Each state has a fixed loop budget. The CC3200 transitions through boot, sensor calibration, cloud mission fetch, preparation countdown, active round (scoring), cloud judging, result sync, and end screen.

01

Boot

Connect WiFi, verify TLS, initialize peripherals.

02

Calibrate

Baseline ultrasonic distance, light level, and temperature.

03

Mission

POST shadow request, Lambda generates wave JSON, poll for ready.

04

Prep

Countdown for players to ready positions.

05

Active

Threat sweep, shield tracking, scoring ticks, attack modes.

06

Judge

Freeze scores, display winner.

07

Sync

POST round results to shadow, Lambda scores and writes S3 artifacts.

08

End

Show final scores, wait for restart or sync retry.

System Architecture

CC3200 Main Controller

  • Global state machine and game loop
  • AWS IoT TLS client, shadow sync, HTTPS mission fetch
  • SPI OLED rendering (radar HUD, scores, state indicators)
  • IR decoding via GPIO interrupt + SysTick timing
  • UART1 TX/RX to Arduino for control/sensor frames

Arduino UNO Coprocessor

  • ULN2003 stepper motor sweep (512-step radar scan)
  • HC-SR04 ultrasonic sampling with median-of-3 filter
  • SG90 servo shield actuation
  • RGB LED, buzzer, joystick, and photoresistor reads
  • SoftwareSerial 9600 baud frame exchange

UART Frame Protocol

Sensor frame (UNO to CC3200): $S,<ms>,<sector>,<dist_cm>,<lux>,<tilt>,<temp_c10>,<hum_10>,<aux>,<joy>\n

Control frame (CC3200 to UNO): $C,<servo_deg>,<step_mode>,<buzz_mode>,<rgb_code>,<state>\n

Implementation

Threat Detection

The UNO sweeps a stepper motor across 512 steps mapped to 16 sectors. At each sensor sample (every 120ms), distance, light, and tilt readings are fused into a 0-15 threat level. The CC3200 applies difficulty scaling, attack mode bonuses, and baseline delta comparisons from the calibration phase.

Shield and Scoring

The defender positions a shield sector using the joystick (analog X mapped to 0-15 sectors with hysteresis). If the shield sector is within 2 of the threat sector, the defender scores proportional to threat level. Otherwise the attacker scores. Scoring ticks happen every 12 main loop iterations.

Attack Modes

The attacker triggers timed effects via IR buttons: Pulse (+2 threat for 140 loops), Jam (shifts threat sector by 3 for 170 loops), and Blind (inflates lux delta by 220 for 170 loops). These stack with base threat computation.

Cloud Pipeline

The CC3200 opens a TLS socket to the AWS IoT HTTPS endpoint on port 8443. It POSTs a MISSION_REQUEST shadow update with the current difficulty. Lambda generates a wave script and writes it to S3, then updates the shadow with MISSION_READY. After the round, a ROUND_DONE POST triggers Lambda to compute weighted scores, build a replay artifact, and update the S3 leaderboard.

OLED Radar HUD

The 128x128 OLED renders a circular radar with sector markers, a sweep line tracking the current stepper position, a threat dot (yellow/red by severity), a shield dot (green), and a threat meter bar on the right. A header bar shows state, difficulty, and cloud status. A footer bar shows contextual instructions per state. All regions use dirty-flag rendering to minimize SPI writes.

Reproducibility

Firmware source compiles with ARM Compiler TI 20.2.7.LTS against CC3200 SDK 1.5.0. Arduino sketches compile in Arduino IDE 2.x. Lambda handler deploys as a single Python 3.12 file. Pin assignments, UART frame formats, and AWS resource names are documented in the project repository.
