DRIVE

Manual control and immediate response

Direct operator control through IR input during manual mode.

  • Directional control from remote buttons
  • Speed adjustment from remote keys
  • Live OLED mode and proximity feedback

AVOID

Deterministic obstacle avoidance loop

Front-threshold gate with left-right comparison and safe recovery.

  • Forward scan with front threshold gate
  • Left-right clearance comparison
  • Reverse and rescan when blocked

PARK

Parallel parking state machine

Cloud-guided park sequence with local fallback when needed.

  • Right-side wall following and gap detection
  • Guidance request before park maneuver
  • Sensor-based fine adjustment at completion

CLOUD

AWS telemetry and guidance pipeline

Telemetry, cloud guidance, and collision alerting in one flow.

  • Device shadow updates for mode and distances
  • Lambda guidance generation and S3 retrieval
  • SNS alert on collision threshold trigger

PROCESS VISUALIZATION

How ParkPilot executes from input to motion

Input Layer

  1. 01Mode select from IR command
  2. 02Sensor stream from Arduino + BMA222

Control Layer

  1. 03State machine decision on CC3200
  2. 04UART command to steering and motor PWM

Cloud Layer

  1. 05IoT shadow sync for mode and distances
  2. 06Lambda guidance and SNS alert feedback

Demo Sequence

  1. Manual control check
  2. Autonomous obstacle avoidance run
  3. Self parking pass
  4. Telemetry and collision safety check

System Split

  • CC3200: mode state machine, parking logic, OLED, WiFi telemetry
  • Arduino: ultrasonic reads, UART frames, steering + motor PWM
  • Sensor frame: $S,<front>,<rear>,<left>,<right>\n
  • Motor command: $M,<speed_pct>,<steer_angle>\n

Proof Targets

  • 3+ clean auto-avoid runs
  • 8/10 successful parallel parking
