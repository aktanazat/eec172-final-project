1. Description

ParkPilot is an RC car controlled by a CC3200 that avoids obstacles, parallel-parks itself, and sends telemetry to AWS. An IR remote selects between manual drive, autonomous obstacle avoidance, and self-parking. Four ultrasonic sensors provide proximity data rendered as a radar map on a 128x128 OLED, while an Arduino co-processor handles sensor reads and motor actuation over UART.

AWS IoT is used for telemetry and parking guidance. Device Shadow stores sensor distances, mode, and speed. IoT Rule triggers Lambda to generate parking guidance in S3. A separate SNS rule emails collision alerts when the BMA222 accelerometer detects impact.

Existing hobby autonomous cars often rely on camera pipelines and heavier compute. ParkPilot uses ultrasonic ranging and classical control on constrained hardware with cloud-assisted guidance.

2. Design

2.1 Functional Specification

The system operates in four modes selected by IR remote, with collision detection running in parallel across all active modes.

IDLE

Car stationary. OLED shows WiFi status, battery, and mode prompt.

MANUAL

IR directional control and speed control with live radar on OLED.

AUTO AVOID

Forward drive with threshold-based steering toward open clearance.

SELF PARK

Gap detection, cloud guidance request, then parallel parking sequence.

Parking Sequence

Pull forward past gap, cut wheel and reverse in, straighten and pull forward, then fine-adjust with sensor feedback.

Collision Detection

BMA222 runs in parallel during active modes. Threshold spike triggers emergency stop, red OLED flash, and SNS email through Shadow update.

2.2 System Architecture

CC3200 Main MCU

  • Mode management and parking state machine
  • SPI OLED radar rendering
  • I2C BMA222 collision monitor
  • WiFi telemetry to AWS IoT and Shadow
  • UART command and sensor frame exchange

Arduino Co-processor

  • 4x HC-SR04 reads using pulseIn at about 20 Hz
  • UART sensor packet stream to CC3200
  • PWM generation for steering servo and L293D
  • Real-time motor command execution

Sensor Frame

$S,<front_cm>,<rear_cm>,<left_cm>,<right_cm>\n

Motor Command

$M,<speed_pct>,<steer_angle>\n

Communication Protocols

  • SPI: CC3200 to OLED
  • I2C: CC3200 to BMA222
  • UART: CC3200 to Arduino
  • GPIO Interrupt: IR receiver to CC3200
  • HTTPS/REST: CC3200 to AWS

Sensors

  • HC-SR04 x4: front, rear, left, right proximity
  • TSOP381xx IR receiver x1: remote control input
  • BMA222 accelerometer x1: collision detection

3. Implementation Goals

Minimal Goals

  • Car assembly with motors, battery, and Arduino
  • Ultrasonic readout and UART transmission
  • CC3200 UART parsing and OLED distance display
  • IR mode switching and manual drive
  • Autonomous obstacle avoidance
  • AWS Shadow telemetry during demo
  • Standalone boot from flash

Target Goals

  • Real-time top-down OLED radar map
  • BMA222 collision-triggered SNS alert
  • All three modes switchable by IR
  • Self parking with Lambda guidance
  • Two-minute live demo sequence
  • Clean wiring and packaging

Stretch Goals

  • Lambda parking arc trajectory and S3 guidance image
  • OLED overlay of fetched guidance image
  • Speed-proportional avoidance behavior
  • Buffered telemetry replay after reconnect

Verification Metrics

  • Obstacle course completion: 3 or more clean runs
  • Parking success rate: 8/10 in standard slot
  • Collisions during scripted demo: 0
  • Mode switch response: under 300 ms
  • Sensor update freshness: under 100 ms
